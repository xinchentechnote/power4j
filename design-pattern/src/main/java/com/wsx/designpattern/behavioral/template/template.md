定义:
    定义了一个算法的骨架，并允许子类为一个或多个步骤提供实现
    模板方法使得子类可以在不改变算法结构的情况下，重新定义算法的某些步骤

使用场景：
    一次性实现一个算法的不变的部分，并将可变的行为留给子类来实现
    个子类中公共的行为被提取出来并集中到一个公共父类中，从而避免代码重复

优点：
    挺高复用性
    提高拓展性
    符合开闭原则
    
缺点：
    类数量增加
    增加系统实现的复杂度
    集成关系自身缺点，如果父类添加新的抽象方法，所有子类都要改一遍
